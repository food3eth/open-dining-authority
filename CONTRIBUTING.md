# Contributing to Open Dining Authority

This project values clarity, auditability, and good faith.

Please open an issue before submitting major pull requests, especially for changes to governance documents.

---

## What You Can Contribute

### Governance & Methodology
- Improvements to review structure
- Reputation rule refinements
- Aggregation edge cases and abuse analysis

### Specifications
- Data model clarifications
- API contract refinements
- Example fixtures for testing

### Reference Implementation
- Validation logic
- Aggregation engine
- Explainability tooling

### Documentation
- Diagrams, onboarding, FAQs

---

## How to Contribute

1. **Open an issue** describing the problem or proposal  
2. Keep PRs **small and focused** (one conceptual change)  
3. For governance changes, include:
   - Problem statement
   - Example scenarios
   - Trade-offs
   - Expected impact

---

## Versioning Rules

Changes that materially affect outputs must:
- Update the relevant versioned document
- Include a changelog note
- Avoid silent behavior changes

---

## Code Standards (if contributing code)

- Prefer deterministic behavior over heuristics
- Prefer explicit configuration over magic defaults
- Tests for aggregation logic are strongly encouraged

---

## Conduct

- Be respectful
- Critique ideas, not people
- No harassment, no spam

Violations may result in removal from the project.
